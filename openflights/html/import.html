OpenFlights: Import data

OpenFlights can import two types of data:

FlightMemory (.html) — FlightMemory does not have an export function, but OpenFlights can "screen-scrape" its FlightData pages. Log into your FlightMemory account, go to the FlightData page, and select "Save Web Page" in your browser. Upload the resulting HTML file. Repeat once for each page of data.

Warning: Be sure to set your FlightMemory language to English before saving the pages.

Note: The airline names used on FlightMemory do not always match those used by OpenFlights, but the import tool will try its best to match them. Importing accuracy is much higher if your FlightData also includes flight numbers; for example, there are four airlines named "Virgin", but only Virgin Atlantic has the code "VS".

Note: There is no duplicate detection at this time. If you import a FlightMemory file twice, all flights in it will be added twice.

Note: The FlightMemory format changes from time to time. Please report any bugs or oddities, and be sure to include the exact error and the "Tmpfile" value from the top of the import page.

OpenFlights (.csv) — Comma-separated value, exported from OpenFlights ("Export" or "Backup") and easily edited or created in Excel or any other spreadsheet. See format specification.
